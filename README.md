# Notes-for-Quants

## Overview

I’m a sophomore at Indiana University, and I built this repository to make quantitative mathematics more accessible and less opaque. Too often, this field feels gatekept—filled with fragmented resources, unexplained jumps in difficulty, and an implicit assumption that you already know how everything fits together.

This repository is my attempt to change that.

It is a comprehensive and structured collection for learning quantitative mathematics from first principles to advanced applications.

## How This Repository Works

The material is intentionally sequential.

It begins with foundational subjects—calculus, linear algebra, and basic probability—and progressively extends into more advanced areas such as stochastic processes, optimization, statistical inference, and quantitative applications.

You are expected to go chapter by chapter, in order.

This is not meant to be skimmed. Each section builds on the last, and skipping ahead will likely create gaps in understanding. The structure is cumulative, so progression should be deliberate.

## Structure

The repository broadly follows this path:

### Foundations

* Calculus (limits, derivatives, integrals)
* Linear Algebra (vectors, matrices, eigenvalues)
* Basic Probability

### Intermediate Topics

* Multivariable Calculus
* Probability Distributions
* Statistics and Inference
* Numerical Methods

### Advanced Topics

* Stochastic Processes
* Optimization
* Measure Theory (more rigorous path)
* Applications in Quantitative Finance and Machine Learning

Each chapter includes:

* Conceptual explanations
* Formal derivations
* Worked examples
* Problem sets
* Code implementations (where relevant)

## Philosophy

This repo is built around two ideas:

* **Rigor** — mathematical precision and logical consistency
* **Intuition** — deep, conceptual understanding

The goal is not just to expose you to material, but to make it genuinely understandable and usable.

## Final Note

This is meant to be worked through slowly. Quantitative mathematics is cumulative, and real understanding comes from consistency, not speed.

If this field has ever felt inaccessible, that’s exactly what this repository is trying to fix.
